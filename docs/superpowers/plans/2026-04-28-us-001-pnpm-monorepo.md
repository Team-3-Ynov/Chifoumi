# US-001 PNPM monorepo skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a minimal PNPM workspace layout at the repo root so `pnpm install` resolves all `apps/*` and `packages/*` stubs and acceptance criteria for GitHub #1 are met.

**Architecture:** Configuration lives in the repository root (`package.json`, `pnpm-workspace.yaml`, `.npmrc`). Each future app and package is a workspace member with a stub `package.json` only—no source, scripts, or dependencies until later tickets. Reproducibility comes from committing `pnpm-lock.yaml` after the first successful install.

**Tech stack:** Node.js ≥ 20, PNPM 9.x (pinned via `packageManager`), JSON manifests only for this ticket.

**Authoritative spec:** `docs/superpowers/specs/2026-04-28-us-001-pnpm-monorepo-design.md`
